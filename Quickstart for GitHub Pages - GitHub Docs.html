إنشاء موقع على GitHub Pages

يمكنك إنشاء موقع GitHub Pages في مستودع جديد أو موجود.

من يمكنه استخدام هذه الميزة؟

صفحات GitHub متاحة في المستودعات العامة مع GitHub Free وGitHub Free للمؤسسات، وفي المستودعات العامة والخاصة مع GitHub Pro وGitHub Team وGitHub Enterprise Cloud وGitHub Enterprise Server. لمزيد من المعلومات، راجع خطط GitHub .

إنشاء مستودع لموقعك

يمكنك إما إنشاء مستودع أو اختيار مستودع موجود لموقعك.

إذا كنت ترغب في إنشاء موقع على صفحات GitHub لمستودع لا ترتبط جميع ملفاته به، فيمكنك تهيئة مصدر نشر لموقعك. على سبيل المثال، يمكنك تخصيص فرع ومجلد لحفظ ملفات مصدر موقعك، أو استخدام سير عمل GitHub Actions مخصص لإنشاء ملفات مصدر موقعك ونشرها.

إذا كان الحساب الذي يملك المستودع يستخدم GitHub Free أو GitHub Free للمؤسسات، فيجب أن يكون المستودع عامًا.

إذا كنت تريد إنشاء موقع في مستودع موجود، فانتقل إلى قسم إنشاء موقعك .

  1. في الزاوية العلوية اليمنى من أي صفحة، حدد ، ثم انقر فوق مستودع جديد .

  2. استخدم القائمة المنسدلة "المالك" لتحديد الحساب الذي تريد أن يمتلك المستودع من خلاله.

  3. اكتب اسمًا لمستودعك ووصفًا اختياريًا. إذا كنت تُنشئ موقعًا لمستخدم أو مؤسسة، فيجب أن يكون اسم مستودعك <user>.github.ioأو <organization>.github.io. إذا كان اسم المستخدم أو المؤسسة يحتوي على أحرف كبيرة، فيجب أن يكون صغيرًا. لمزيد من المعلومات، راجع " ما هي صفحات GitHub؟" .

  4. اختر رؤية المستودع. لمزيد من المعلومات، راجع " حول المستودعات" .

  5. حدد تهيئة هذا المستودع باستخدام ملف README .

  6. انقر فوق إنشاء مستودع .

إنشاء موقعك

قبل إنشاء موقعك، يجب أن يكون لديك مستودع لموقعك على GitHub. إذا لم تكن تنشئ موقعك في مستودع موجود، فراجع إنشاء مستودع لموقعك .

تحذير

مواقع صفحات GitHub متاحة للعامة على الإنترنت، حتى لو كان مستودع الموقع خاصًا (إذا سمحت خطتك أو مؤسستك بذلك). إذا كانت لديك بيانات حساسة في مستودع موقعك، فقد ترغب في إزالتها قبل النشر. لمزيد من المعلومات، راجع قسم " حول المستودعات" .

  1. على GitHub، انتقل إلى مستودع موقعك.

  2. حدد مصدر النشر الذي تريد استخدامه. راجع تهيئة مصدر النشر لموقع صفحات GitHub الخاص بك .

  3. أنشئ ملف الإدخال لموقعك. ستبحث صفحات GitHub عن ملف index.htmlأو index.mdأو README.mdكملف إدخال لموقعك.

    إذا كان مصدر النشر لديك فرعًا ومجلدًا، فيجب أن يكون ملف الإدخال في أعلى مستوى مجلد المصدر على فرع المصدر. على سبيل المثال، إذا كان مصدر النشر لديك هو /docsالمجلد على mainالفرع، فيجب أن يكون ملف الإدخال موجودًا في /docsمجلد فرع يُسمى main.

    إذا كان مصدر النشر لديك سير عمل GitHub Actions، فيجب أن يتضمن المنتج الذي تنشره ملف الإدخال في أعلى مستوى له. بدلاً من إضافة ملف الإدخال إلى مستودعك، يمكنك اختيار أن يقوم سير عمل GitHub Actions بإنشاء ملف الإدخال الخاص بك عند تشغيله.

  4. قم بتكوين مصدر النشر الخاص بك. راجع تكوين مصدر النشر لموقع GitHub Pages الخاص بك .

  5. تم بناء موقع صفحات GitHub الخاص بك ونشره باستخدام سير عمل GitHub Actions. لمزيد من المعلومات، راجع "عرض سجل تشغيل سير العمل" .

    ملحوظة

    GitHub Actions مجاني للمستودعات العامة. تُطبق رسوم استخدام على المستودعات الخاصة والداخلية التي تتجاوز الحد الشهري للدقائق المجانية. لمزيد من المعلومات، راجع "الفوترة والاستخدام" .

عرض موقعك المنشور

  1. تحت اسم مستودعك، انقر على "الإعدادات ". إذا لم تظهر لك علامة تبويب "الإعدادات"، فاختر القائمة المنسدلة، ثم انقر على "الإعدادات" .

  2. في قسم "الترميز والأتمتة" في الشريط الجانبي، انقر فوق الصفحات .

  3. لرؤية موقعك المنشور، ضمن "صفحات GitHub"، انقر فوق زيارة الموقع .

ملحوظة

قد يستغرق نشر تغييرات موقعك ما يصل إلى ١٠ دقائق بعد نشرها على GitHub. إذا لم تظهر تغييرات موقعك على GitHub Pages في متصفحك بعد ساعة، فراجع " حول أخطاء بناء Jekyll لمواقع GitHub Pages" .

  • إذا كنت تنشر من فرع ولم ينشر موقعك تلقائيًا، فتأكد من قيام شخص لديه أذونات المسؤول وعنوان بريد إلكتروني تم التحقق منه بالدفع إلى مصدر النشر.
  • يتم دفع الالتزامات بواسطة سير عمل GitHub Actions الذي يستخدم GITHUB_TOKENعدم تشغيل بناء GitHub Pages.

مولدات المواقع الثابتة

تنشر صفحات GitHub أي ملفات ثابتة ترسلها إلى مستودعك. يمكنك إنشاء ملفاتك الثابتة الخاصة أو استخدام مُولّد مواقع ثابتة لبناء موقعك. كما يمكنك تخصيص عملية البناء الخاصة بك محليًا أو على خادم آخر.

إذا كنت تستخدم عملية بناء مخصصة أو مُولِّد مواقع ثابتة غير Jekyll، فيمكنك كتابة سير عمل GitHub Actions لبناء موقعك ونشره. يوفر GitHub قوالب سير عمل لعدة مُولِّدات مواقع ثابتة. لمزيد من المعلومات، راجع تهيئة مصدر نشر لموقع GitHub Pages الخاص بك .

إذا نشرت موقعك من فرع مصدر، فستستخدم صفحات GitHub Jekyll لبناء موقعك افتراضيًا. إذا كنت ترغب في استخدام مُولّد مواقع ثابتة غير Jekyll، فننصحك بكتابة إجراءات GitHub لبناء موقعك ونشره. وإلا، عطّل عملية بناء Jekyll بإنشاء ملف فارغ .nojekyllفي جذر مصدر النشر، ثم اتبع تعليمات مُولّد المواقع الثابتة لبناء موقعك محليًا.

ملحوظة

لا تدعم صفحات GitHub لغات جانب الخادم مثل PHP أو Ruby أو Python.

أنواع MIME على صفحات GitHub

نوع MIME هو ترويسة يرسلها الخادم إلى المتصفح، موفرًا معلومات حول طبيعة وتنسيق الملفات التي يطلبها المتصفح. يدعم GitHub Pages أكثر من 750 نوع MIME عبر آلاف امتدادات الملفات. قائمة أنواع MIME المدعومة مُولّدة من مشروع mime-db .

مع أنه لا يمكنك تحديد أنواع MIME مخصصة لكل ملف أو مستودع، يمكنك إضافة أو تعديل أنواع MIME لاستخدامها في صفحات GitHub. لمزيد من المعلومات، راجع إرشادات المساهمة في قاعدة بيانات MIME .

الخطوات التالية

يمكنك إضافة المزيد من الصفحات إلى موقعك بإنشاء المزيد من الملفات الجديدة. سيتوفر كل ملف على موقعك في نفس بنية المجلد كمصدر النشر. على سبيل المثال، إذا كان مصدر النشر لموقع مشروعك هو الفرع gh-pages، وقمت بإنشاء ملف جديد باسمه /about/contact-us.mdعلى gh-pagesالفرع، فسيكون الملف متاحًا على https://<user>.github.io/<repository>/about/contact-us.html.

يمكنك أيضًا إضافة قالب لتخصيص مظهر موقعك. لمزيد من المعلومات، راجع إضافة قالب إلى صفحات GitHub باستخدام Jekyll .
